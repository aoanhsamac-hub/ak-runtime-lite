## Key Requirements (from available documentation)

### Economic Principles

- All capabilities must demonstrate economic value
- No capability exists without usage measurement
- ROI must be tracked for every operational capability
- Treasury impact must be assessed before activation

### Mandatory Metrics

Every capability must track:
- usage_count: number of times invoked
- total_value: estimated value generated
- total_cost: estimated cost incurred
- roi: value / cost ratio
- adoption_status: not_tracked / tracking / adopted / deprecated

### Capability Lifecycle Value

| Stage | Economic Requirement |
|---|---|
| DISABLED | No tracking required |
| TRACKING | Usage + cost measurement |
| ADOPTED | ROI demonstrated, value tracked |
| EVOLVED | Positive ROI, continuous improvement |
| DEPRECATED | Usage ended, archived |

### Treasury Impact Assessment

Before any activation state change to PILOT_ACTIVE or higher:
- Cost projection must be completed
- Budget allocation must be confirmed
- ROI threshold must be met
- Iris must provide economic assessment
