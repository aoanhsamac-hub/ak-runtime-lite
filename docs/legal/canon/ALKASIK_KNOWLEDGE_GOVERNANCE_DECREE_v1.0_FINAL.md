## Key Requirements (from available documentation)

### Knowledge Lifecycle

```
Evidence
   ↓
Lesson
   ↓
Knowledge
   ↓
Skill
   ↓
Capability
```

- Evidence must be classified per I0-I9 scale
- Lessons must be distilled from evidence
- Knowledge must be validated from lessons
- Skills must be promoted from validated knowledge
- Capabilities must be built from promoted skills

### Knowledge Governance Rules

- No evidence → lesson without quality check
- No lesson → knowledge without Sage review
- No knowledge → skill without governance gate
- No skill → capability without Janus authorization
- No autonomous promotion

### Knowledge Classes

| Class | Retention | Description |
|---|---|---|
| TRANSIENT | 30 days | Temporary observations |
| OPERATIONAL | 1 year | Active operational knowledge |
| CANONICAL | Permanent | Official national knowledge |
| ARCHIVAL | Permanent | Historical record |

### Mandatory Metadata

Every knowledge record must have:
- source_type (evidence/lesson/knowledge/skill/capability)
- source_id
- owner_agent
- reviewer_agent
- knowledge_class (TRANSIENT/OPERATIONAL/CANONICAL/ARCHIVAL)
- created_at
- retention_until
